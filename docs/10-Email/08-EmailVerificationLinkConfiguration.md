# Email Verification Link Configuration

How to configure the email verification link URL that users receive in their email.

---

## Overview

When users request email verification, they receive an email with a verification link:

```
https://yourdomain.com/verify-email?token=bzyF3tWA17PgewuRy4iyR4nVQmfF9jWo2BvEu5pYZfU
```

The domain part (`https://yourdomain.com/verify-email`) is **configurable** via environment variable.

---

## Configuration

### Environment Variable

```
Platform__EmailVerificationRedirectUrl=https://yourdomain.com/verify-email
```

### By Environment

#### Development (Local .env)
```env
Platform__EmailVerificationRedirectUrl=http://localhost:3000/verify-email
```

#### Production (Render/Heroku/Docker)
```env
Platform__EmailVerificationRedirectUrl=https://flow.kromic.in/verify-email
```

#### Staging
```env
Platform__EmailVerificationRedirectUrl=https://staging.flow.kromic.in/verify-email
```

---

## How It Works

### Email Generation Flow

```
1. User sends email verification request
   POST /api/v1/users/verify-email
   { "email": "user@example.com" }

2. Backend generates token (32 random bytes)
   Token: bzyF3tWA17PgewuRy4iyR4nVQmfF9jWo2BvEu5pYZfU

3. Backend creates verification link using config
   Config: Platform__EmailVerificationRedirectUrl
   Link: {config}?token={token}
   Result: https://flow.kromic.in/verify-email?token=bzyF3tWA17PgewuRy4iyR4nVQmfF9jWo2BvEu5pYZfU

4. Backend sends email with link

5. User clicks link in email
   Browser navigates to: https://flow.kromic.in/verify-email?token=...
   (Frontend handles from here)
```

## Frontend Handling

### Page: /verify-email

This page receives the token from URL:

```javascript
// Extract token from URL
const urlParams = new URLSearchParams(window.location.search);
const token = urlParams.get('token');

// Send to backend for verification
const response = await fetch('/api/v1/users/verify-email-token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${accessToken}`
  },
  body: JSON.stringify({ token })
});

if (response.ok) {
  // Show success page
  showSuccess("Email verified!");
  // Redirect to dashboard after 2 seconds
  setTimeout(() => redirect('/dashboard'), 2000);
}
```

## Important Notes

### URL Must Be Valid

- ✅ Full URL: `https://flow.kromic.in/verify-email`
- ✅ With port: `http://localhost:3000/verify-email`
- ✅ With path: `https://flow.kromic.in/auth/verify-email`
- ❌ Partial: `/verify-email` (won't work in email)
- ❌ No protocol: `flow.kromic.in/verify-email` (won't work)

### Token Handling

- Token is generated by **backend** (cryptographically secure)
- Token is appended to URL as query parameter: `?token=...`
- Token is **NOT in the domain** - it's in the path/query
- Token is **case-sensitive**
- Token **expires after 24 hours**

### One-Time Use

```
1. User clicks link with token
2. Backend validates token
3. Backend marks email as verified
4. Backend clears token from database
5. If user clicks link again, gets "Token already used" or "Invalid token"
```

    "DefaultPlanCode": "free",

    "EmailVerificationRedirectUrl": "https://flow.kromic.in/verify-email"
  }
}

## Frontend Route Setup

### React Router Example

```typescript
import { useEffect, useState } from 'react';
import { useSearchParams } from 'react-router-dom';

export function VerifyEmailPage() {
  const [searchParams] = useSearchParams();
  const [status, setStatus] = useState('loading');
  const token = searchParams.get('token');

  useEffect(() => {
    if (!token) {
      setStatus('error');
      return;
    }

    verifyToken(token)
      .then(() => setStatus('success'))
      .catch(err => {
        console.error(err);
        setStatus('error');
      });
  }, [token]);

  if (status === 'loading') return <div>Verifying...</div>;
  if (status === 'success') return <div>✅ Email verified!</div>;
  return <div>❌ Verification failed</div>;
}
```

### Route Configuration

```typescript
<Route path="/verify-email" element={<VerifyEmailPage />} />
```

## Troubleshooting

### Issue: Email link is broken

**Solution:** Check `Platform__EmailVerificationRedirectUrl` is set correctly

```bash
# Check value
echo $Platform__EmailVerificationRedirectUrl
```

### Issue: Users get "Invalid domain" error

**Solution:** Verify URL is accessible from the internet

```bash
curl https://flow.kromic.in/verify-email?token=test
```

Should return your frontend page (not 404).

### Issue: Token not in URL

**Solution:** Check frontend is reading query parameter correctly

```javascript
const token = new URLSearchParams(window.location.search).get('token');
console.log(token); // Should show token
```
